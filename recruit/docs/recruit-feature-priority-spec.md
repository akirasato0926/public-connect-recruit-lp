# Recruit 追加機能優先仕様

## 概要

本ドキュメントは、`Recruit` の既存要件定義書とユーザーストーリー一覧をもとに、
官公庁向け機能と求職者向け機能をまたぐ追加機能の優先順位と仕様ドラフトを整理したものです。

優先度は以下の順で定義します。

1. 採用ファネルダッシュボード
2. 母集団形成シミュレーター
3. 流入・コンテンツ効果分析と求職者レコメンド連携

## 既存ストーリー横断マップ

### 官公庁側

#### A. 採用計画

- `採用目標人数を登録・確認する`
- `採用目標人数をダッシュボードで確認する`
- `過去の採用ファネルデータを参照する`
- `必要エントリー数（目安）を確認する`
- `試験日程を登録・確認する`

#### N. 選考管理

- `全選考フェーズの受験者状況をダッシュボードで一覧確認する`
- `応募者ごとに全選考結果を統合して閲覧する`
- `選考フェーズごとの通過率・離脱率をグラフで確認する`
- `応募者へのアンケートを送付する`

#### B. 採用広報

- `コンテンツごとの閲覧数・視聴完了率・流入経路をダッシュボードで確認する`
- `採用案内ごとの閲覧数・クリック数・応募転換率を確認する`
- `フォロワー属性・開封率・クリック率を確認する`
- `フォロワー数の推移をグラフで確認し施策との相関を分析する`
- `AIが関連コンテンツをレコメンド表示する`

### 求職者側

- `価値観・思考性に関する設問に回答しプロフィールを蓄積する`
- `AIが最適な地域・求人・コンテンツのレコメンドを受け取る`
- `閲覧・応募・イベント参加等のアクティビティ履歴を記録・活用する`
- `普段接触しない地域・職種の情報を探索型表示で受け取る`

## 共通データレイヤー

- 求人単位の採用目標人数
- 求人単位の選考フェーズ別人数
- 求職者の閲覧履歴
- 求職者の応募履歴
- 求職者のイベント参加履歴
- コンテンツ別の流入元、クリック数、応募転換数
- 自治体フォロー、通知開封、クリック履歴

## 優先1: 採用ファネルダッシュボード

### 目的

採用担当者が求人ごとの計画値と実績値を同じ画面で比較し、
打ち手が必要な求人や選考フェーズを早期に把握できるようにする。

### 主な利用者

- `EMP_ADMIN`
- `EMP_MANAGER`
- `EMP_RECRUITMENT`
- 閲覧専用として `RP`

### 主要指標

- 採用目標人数
- エントリー数
- 一次通過率
- 二次通過率
- 内定数
- 承諾数
- 充足率
- フェーズ別離脱率
- 平均滞留日数

### 受入基準ドラフト

- 求人ごとに `目標N名 / 承諾N名 / 充足率N%` を表示できること
- 全体合計と求人別の両方を切り替えて確認できること
- フェーズ別通過率と離脱率をグラフ表示できること
- 期間を `年度 / 月次 / 募集回` 単位で切り替えられること
- 充足率が閾値未満の求人をアラート表示できること
- 過去年度との比較差分を表示できること

## 優先2: 母集団形成シミュレーター

### 目的

採用目標人数に対して必要なエントリー数や各フェーズ通過人数の目安を逆算し、
採用計画を感覚ではなくデータに基づいて立てられるようにする。

### 受入基準ドラフト

- 採用目標人数を入力すると過去平均通過率を用いて必要人数を自動計算できること
- 過去実績の参照年度を切り替えられること
- 過去データが不足する場合は `データ蓄積中` を表示すること
- 計算結果が参考値であることを明示する注記を表示すること

## 優先3: 流入・コンテンツ効果分析と求職者レコメンド連携

### 目的

官公庁には広報施策の成果を返し、
求職者には関心に近い求人・地域・コンテンツを継続的に提示する。

### 受入基準ドラフト

- コンテンツ別の閲覧数、流入元、クリック数、応募転換率を確認できること
- 求職者の閲覧・応募・イベント参加履歴を記録できること
- 求職者ごとに求人、地域、記事のレコメンドを表示できること
- 関連度が高い推薦と探索型推薦を分けて表示できること
- 官公庁側では広報施策ごとの成果比較ができること
