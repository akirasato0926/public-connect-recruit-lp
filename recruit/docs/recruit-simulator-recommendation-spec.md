# 母集団形成シミュレーターとレコメンド連携仕様ドラフト

## 機能1: 母集団形成シミュレーター

### 目的

採用目標から必要母集団を逆算し、感覚ではなくデータに基づく採用計画を可能にする。

### 主な利用者

- `EMP_ADMIN`
- `EMP_MANAGER`
- `EMP_RECRUITMENT`

### 入力

- 採用目標人数
- 対象求人
- 対象年度
- 過去の通過率データ

### 出力

- 必要エントリー数
- 必要一次通過数
- 必要二次通過数
- 必要内定数
- 必要承諾数

### 受入基準

- 採用目標人数を入力すると過去平均通過率から必要人数を計算できること
- 年度ごとに参照実績を切り替えられること
- 過去データが不足する場合は `データ蓄積中` と表示できること
- 計算結果が参考値である注記を表示できること
- ダッシュボード上の実績と並べて比較できること

## 機能2: 流入・コンテンツ効果分析

### 主な利用者

- 官公庁担当者
- `PC_運営者`

### 指標

- 閲覧数
- 視聴完了率
- 流入元
- クリック数
- 応募転換率
- 開封率
- クリック率

### 受入基準

- コンテンツ別に閲覧数、流入元、クリック数、応募転換率を確認できること
- フォロワー通知について開封率とクリック率を確認できること
- 施策ごとに比較表示できること
- 官公庁単位と求人単位で集計軸を切り替えられること

## 機能3: 求職者レコメンド連携

### 主な利用者

- `JS_求職者`

### 利用データ

- 基本情報
- 価値観プロフィール
- 閲覧履歴
- 応募履歴
- イベント参加履歴
- フォロー履歴

### 推薦対象

- 求人
- 地域
- 記事
- 動画
- 採用案内

### 推薦モード

#### 関連度ベース

過去の閲覧、応募、プロフィールに近い対象を提示する。

#### 探索型

普段接触していない地域、職種、コンテンツを意図的に混ぜて提示する。

### 受入基準

- 求職者ごとに求人、地域、記事のレコメンドを表示できること
- 関連度ベース推薦と探索型推薦を分けて表示できること
- 閲覧、応募、イベント参加履歴を記録できること
- 官公庁側の分析指標として求職者行動を匿名集計で利用できること
