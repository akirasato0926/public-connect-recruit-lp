# Recruit 既存ストーリー横断マップ

## 目的

官公庁向けの採用計画・選考管理・採用広報と、
求職者向けの行動履歴・レコメンドを接続し、
追加機能をどこに乗せるべきかを明確にする。

## 官公庁向けストーリー

### A. 採用計画

| 既存ストーリー | 今回の追加機能との接続 |
| --- | --- |
| 採用目標人数を登録・確認する | ファネルダッシュボードの基準値 |
| 採用目標人数をダッシュボードで確認する | 充足率、進捗率の表示先 |
| 過去の採用ファネルデータを参照する | シミュレーターの計算根拠 |
| 必要エントリー数（目安）を確認する | 母集団形成シミュレーターの中心機能 |
| 試験日程を登録・確認する | ダッシュボード上の滞留・スケジュール監視 |

### N. 選考管理

| 既存ストーリー | 今回の追加機能との接続 |
| --- | --- |
| 全選考フェーズの受験者状況をダッシュボードで一覧確認する | ファネルダッシュボードの実績表示 |
| 応募者ごとに全選考結果を統合して閲覧する | 滞留分析、離脱分析の根拠データ |
| 選考フェーズごとの通過率・離脱率をグラフで確認する | ファネル分析の可視化軸 |
| 応募者へのアンケートを送付する | 離脱理由、辞退理由分析の拡張余地 |

### B. 採用広報

| 既存ストーリー | 今回の追加機能との接続 |
| --- | --- |
| コンテンツごとの閲覧数・視聴完了率・流入経路を確認する | 効果分析の中心指標 |
| 採用案内ごとの閲覧数・クリック数・応募転換率を確認する | 応募転換分析 |
| フォロワー属性・開封率・クリック率を確認する | ターゲティング施策分析 |
| フォロワー数の推移をグラフで確認し施策との相関を分析する | 施策比較、改善判断 |
| AIが関連コンテンツをレコメンド表示する | 求職者向け推薦との接点 |

## 求職者向けストーリー

| 既存ストーリー | 今回の追加機能との接続 |
| --- | --- |
| 価値観・思考性に関する設問に回答しプロフィールを蓄積する | 推薦モデルの入力情報 |
| AIが最適な地域・求人・コンテンツのレコメンドを受け取る | レコメンドUI本体 |
| 閲覧・応募・イベント参加等のアクティビティ履歴を記録・活用する | 分析基盤、推薦精度向上 |
| 普段接触しない地域・職種の情報を探索型表示で受け取る | 探索型推薦の定義 |

## 追加が必要な共通データ

- 求人ごとの採用目標人数
- 選考フェーズ別の人数と日時
- 求職者の閲覧履歴
- 求職者の応募履歴
- 求職者のイベント参加履歴
- コンテンツ別の流入元、クリック数、応募転換数
- 通知送信、開封、クリックのログ
